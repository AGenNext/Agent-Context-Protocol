Governed context for agent execution.

ACP defines how context is requested, authorized, scoped, signed, delivered, expired, and audited for agent actions.

Core rule

No identity. No contract. No policy. No scoped context.

Why ACP exists

Least context

Agents receive only the context required for a declared action, not broad memory dumps or tenant-wide data.

Runtime governance

Context is bound to identity, contract, policy, tool scope, data scope, expiry, and audit obligations.

Trust evidence

Signed packets create evidence that context was authorized, scoped, delivered, and used under policy.

Protocol flow

1. Verify identity
2. Verify contract
3. Evaluate policy
4. Issue signed packet
5. Expire and audit
